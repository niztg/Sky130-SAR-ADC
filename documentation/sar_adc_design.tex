\section{Introduction and Specifications}
\label{section:intro}
The successive-approximation (SAR) ADC is one of the most widely-used and versatile analog-to-digital converter toplogies \cite{murmann2016sar}.
Owing to its low power consumption, 
low latency and compact form factor, it has proven supremely versatile, with notable usage in IoT devices, medical equipment and microcontrollers. This document describes an original 10-bit SAR ADC designed and verified in the 
SkyWater Sky130 open-source 130nm process, a CMOS PDK developed by SkyWater Technology and Google that provides full device model access without an NDA. 
The design targets the specifications in Table~\ref{tab:specs} and covers the complete front-end flow: behavioral modeling from first principles, transistor-level 
schematic capture in xschem, ngspice simulation against Sky130 device models, and Verilog RTL synthesized to Sky130 standard cells via Yosys. Physical layout 
is out of scope; the project targets schematic-level verification.

The design followed a bottom-up flow, progressing sequentially through each of the following stages:
\begin{enumerate}
    \item First-principles sizing and yield analysis via Python behavioral model
    \item Block-level schematic capture in xschem and isolation verification in ngspice
    \item Verilog RTL synthesis to Sky130 standard cells via Yosys
    \item Full-ADC SPICE integration and static performance characterisation
\end{enumerate}

This document reports the design decisions, block-level verification, integration debugging and static characterisation of the 10-bit SAR ADC. Ultimately, the design achieves a static ENOB of 7.27 bits, falling short of its 9-bit target, attributable to a characterised top-plate feedthrough mechanism detailed in Section~\ref{section:integration}.

\section{Architecture Overview}
\label{section:architecture}

\subsection{Algorithm Description}
The binary code corresponding to $V_\mathrm{in}$ is resolved bit-by-bit from MSB to LSB. Each bit $N$ contributes a weighted voltage $\frac{V_\mathrm{ref}}{2^{10-N}}$ to a running trial voltage $V_\mathrm{analog}$; once all bits are settled, $V_\mathrm{analog} = \frac{W}{1024} V_\mathrm{ref}$, where $W$ is the sum of the retained bit weights. At each step, the trial bit is kept if its inclusion keeps $V_\mathrm{analog} \leq V_\mathrm{in}$, and discarded otherwise. The process repeats until all 10 bits are resolved.

Using an example voltage of $\qty{1.30}{V}$, the behavior of the algorithm can be traced through as follows:

\begin{enumerate}
    \item Assert bit 9 of the trial code, corresponding to an analog voltage of $V_\mathrm{analog} = \frac{1}{2^{10-9}} \cdot V_\mathrm{ref} = \qty{0.9}{V}$
    \item Compare $V_\mathrm{in} = \qty{1.3}{V}$ to $V_\mathrm{analog}$; $V_\mathrm{in} > V_\mathrm{analog}$, therefore retain bit 9, assert bit 8. New trial code: 1100000000, corresponding voltage $V_\mathrm{analog} = \frac{1}{2^{10-9}} \cdot V_\mathrm{ref} + \frac{1}{2^{10-8}} \cdot V_\mathrm{ref} = \qty{1.35}{V}$
    \item Compare $V_\mathrm{in}$ to $V_\mathrm{analog}$; $V_\mathrm{in} < V_\mathrm{analog}$ therefore reject bit 8, assert bit 7. New code = 1010000000, corresponding voltage $V_\mathrm{analog} = \frac{1}{2^{10-9}} \cdot V_\mathrm{ref} + \frac{1}{2^{10-7}} \cdot V_\mathrm{ref} = \qty{1.125}{V}$
\end{enumerate}

This process repeats until all 10 bits are individually checked, in this case resulting in the code $1011100011 = \qty{1.299}{V}$.

\subsection{Operation}
The design is comprised of four major blocks: the bootstrapped switch, CDAC binary-weighted array, StrongARM comparator and SAR logic block, and works in two phases: sampling and conversion. 
During the sampling phase, the bootstrapped switch connects $V_\mathrm{in}$ to all 
bottom plates of the CDAC array simultaneously, while the shared top plate node $V_\mathrm{DAC}$ is reset 
to $V_\mathrm{cm} = V_\mathrm{ref}\, /\,2$. This establishes a charge 
on the CDAC of $Q = (V_\mathrm{ref}/2 - V_\mathrm{in}) \cdot C_{total}$. At the 
start of conversion, the bootstrapped switch is disconnected, leaving $V_\mathrm{DAC}$ floating
while the bottom plate of each node is shorted to ground. The SAR 
logic then performs a binary search from the MSB downward: each trial bit 
steers its corresponding bottom plate to $V_\mathrm{ref}$, shifting $V_\mathrm{DAC}$ upward 
by an amount proportional to that capacitor's weight. At the conclusion of this search, the StrongARM comparator 
evaluates whether $V_\mathrm{ref}/2$ is greater than $V_\mathrm{DAC}$, equivalent to testing whether $V_\mathrm{in}$ exceeds the 
analog value of the current trial code. The SAR logic retains or clears each 
bit accordingly:

\begin{enumerate}
    \item Assert trial code $1000000000$; bottom plate of $512C$ switches to 
    $V_\mathrm{ref}$, shifting $V_\mathrm{DAC}$ upwards to $\frac{512}{1024} V_\mathrm{ref}$.
    \item If $V_\mathrm{DAC} < V_\mathrm{ref}/2$ (equivalently, $V_\mathrm{in} > \frac{512}{1024} V_\mathrm{ref}$), 
    retain bit 9; otherwise clear it and return that bottom plate to ground
    \item Assert the next trial bit (bit 8; adding an upwards shift of $\frac{256}{1024} V_\mathrm{ref}$ to $V_\mathrm{DAC}$) and repeat in descending order until all 
    10 bits are resolved
\end{enumerate}

This process is summarized by the block diagram in figure~\ref{fig:block_diagram}.

\label{section:block_design}
This section covers the transistor-level design of each block in the ADC, including the bootstrapped sampling switch, the charge-redistribution DAC, the StrongARM comparator, and the SAR control logic, with particular attention to the sizing decisions that tie back to the 10-bit, 1\,MS/s specifications.

% ─────────────────────────────────────────────
\subsection{Python Behavioral Model}
\label{subsection:behavioral_model}

The minimum valid unit capacitance, $C_\mathrm{unit}$, that can be used for any capacitor in the CDAC array is set by requiring the rms $kT/C$ sampling noise to remain below half an LSB, giving equation~\ref{eq:thermal_noise_floor}.

\begin{equation}
    C_\mathrm{min} = \frac{kT}{(V_\mathrm{LSB}/2)^2} \label{eq:thermal_noise_floor}
\end{equation}

with $V_\mathrm{LSB} = \frac{V_\mathrm{ref}}{2^N}$, where $N$ is the resolution of the instrument. Substituting $k = \qty{1.38e-23}{\joule\per\kelvin}$, $T = \qty{300}{K}$, $V_\mathrm{ref} = \qty{1.8}{V}$ and $N=10$,
$C_\mathrm{min} $ is found as $\qty{5.36}{\femto F}$.

In order to determine a suitable unit capacitance, a Monte Carlo mismatch analysis was conducted. Twenty candidate values of $C_\mathrm{unit}$ were swept from $3\,C_\mathrm{min}$ to $6\,C_\mathrm{min}$ ($\approx\qty{16.1}{fF}$ to $\approx\qty{32.2}{fF}$), with $C_\mathrm{min}$ itself as the natural lower anchor for the range, since below it thermal noise alone exceeds the half-LSB budget regardless of mismatch. For each candidate value, 2\,000 independent trials were run. In each trial, a binary-weighted capacitor array,
$[512, 256, 128, ..., 2, 1, 1] \times C_\mathrm{unit}$, representing the CDAC array, was generated with every capacitor perturbed according to $\sigma_{C/C} \, / \, \sqrt{n_\mathrm{cells}}$,
where the mismatch $\sigma_{C/C}$ is 2\%, and $n_\mathrm{cells} = C_\mathrm{unit} \, / \, C_\mathrm{min}$. Using this perturbed array, the SAR conversion algorithm was executed on $2^{16} = 65\,536$
input voltages spanning the full input range (64 per binary code), and the DNL and INL were computed from the resulting code histogram, with the maximum $|\mathrm{INL}|$ recorded for that trial.

The yield is computed as the fraction of trials where the maximum INL is less than 0.5 LSB. As can be seen in figure~\ref{fig:yield_vs_cunit}, the yield surpasses its target of 99\%
at approximately $\qty{24}{fF}$. Therefore, to allow for some margin, a target value of $C_\mathrm{unit} = \qty{25}{fF}$ was selected, giving a total CDAC capacitance of $1024 \times \qty{25}{fF} = \qty{25.6}{pF}$.

\subsection{CDAC}
\label{subsection:cdac}

The CDAC is a binary-weighted array of capacitors, $[512, 256, 128, ..., 2, 1, 1] \times C_\mathrm{unit}$. Including the termination capacitor of capacitance 
$C_\mathrm{unit}$, it spans 11 capacitors with a total capacitance of $1024 \cdot C_\mathrm{unit} = \qty{25.6}{pF}$. The top plate of each capacitor is tied to the common node $V_\mathrm{DAC}$,
while each capacitor has a unique bottom plate node.

Sampling is performed on the bottom plates rather than the top plate. The top plate, $V_\mathrm{DAC}$, is the node the comparator reads directly; if sampling charged the top plate instead, $V_\mathrm{DAC}$ would inherit $V_\mathrm{in}$'s settling behavior and any sampling-switch nonidealities directly at the comparator input, with no capacitive division to attenuate them. Driving $V_\mathrm{in}$ onto the bottom plates instead means $V_\mathrm{in}$ only ever appears at $V_\mathrm{DAC}$ through the capacitor network itself, and the top plate is free to be reset to a clean, fixed $V_\mathrm{cm}$ independent of the input. This is also why the comparator never sees $V_\mathrm{in}$ directly (Section~\ref{subsection:strongarm_comparator}): by the time conversion starts, $V_\mathrm{in}$ has already been folded into the charge on the array.

During operation, the CDAC undergoes two stages:

\begin{enumerate}
    \item Sampling: the bottom plate of each capacitor is driven to $V_\mathrm{in}$, while $V_\mathrm{DAC}$ is driven to $V_\mathrm{cm} = V_\mathrm{ref} \, / \, 2$. This locks in a charge of $Q = 1024 \cdot C_\mathrm{unit} \cdot (V_\mathrm{ref} \, / \, 2 - V_\mathrm{in})$ onto the CDAC.
    \item Conversion: $V_\mathrm{DAC}$ is disconnected from its source and becomes floating. The SAR logic block drives the bottom plate of each capacitor to either $V_\mathrm{ref}$ or ground, depending on whether the corresponding bit in the code under trial is high or low. For example, if the code under trial is $1010000000$, then the bottom plates of $C_9 = 512 C_\mathrm{unit}$ and $C_7 = 128 C_\mathrm{unit}$ are sent to $V_\mathrm{ref}$, with the remaining bottom plates being shorted to ground. At the end of conversion, $V_\mathrm{DAC} = \frac{W}{1024} V_\mathrm{ref} + V_\mathrm{ref} \, / \, 2 - V_\mathrm{in}$. Note that during conversion, the bottom plate of the termination capacitor is always shorted to ground.
\end{enumerate}

For each analog voltage under test, sampling occurs once at the start of the procedure, while conversion happens once per bit, progressing from the MSB to the LSB. 
The transistor-level logic that controls the CDAC's behavior is shown in figure~\ref{fig:cdac_cell}.

The diagram in figure~\ref{fig:cdac_cell} applies to all capacitors in the CDAC except for the termination capacitor. Note that for compactness, the transmission gate responsible for charging $V_\mathrm{DAC}$ to
$V_\mathrm{cm}$ during sampling is included in figure~\ref{fig:cdac_cell}; however, because $V_\mathrm{DAC}$ is shared by all capacitors, the entire CDAC only uses one such transmission gate rather than one per cell. 
The signal $cdac\_sw_N$ represents the high/low value of $N^{th}$ bit in the trial code. The capacitor $C_{N}$ is responsible for the voltage contribution of this bit to $V_\mathrm{DAC}$, and has a capacitance
value of $C_N = 2^N \cdot C_\mathrm{unit}$.

During sampling, $\phi_s$ is high, meaning that both transmission gates are active and $V_\mathrm{DAC}$ gets sent to $V_\mathrm{cm}$ while each $V_{bp_N}$ is sent to $V_\mathrm{in}$. 
Thereafter, $\phi_s$ turns off and both transmission gates are switched off, leaving $V_\mathrm{DAC}$ floating and initiating the conversion stage. If $cdac\_sw_N$ is high, then its complement $\overline{\mathit{cdac\_sw}}_N$ is low and the PMOS of the
pull-up network turns on, shorting $bp_N$ to $V_\mathrm{ref}$. Otherwise, the top NMOS in the pull-down network is activated and $bp_N$ is routed to ground. The additional NMOS gated by $\bar{\phi}_s$ in the pull-down
network is introduced due to contention; if it were not there, then the pull-down network would provide a path to ground during sampling while $cdac\_sw_N$ is low. Since $\phi_s$ is guaranteed to be high during sampling and low during conversion, this NMOS ensures that the pull-down network is never active during sampling but can still be active during conversion.

Note that due to its repetitive nature, no schematic for the CDAC array was designed by hand, with $\texttt{cdac.spice}$ instead being written directly.

\subsection{Bootstrapped Switch}
\label{subsection:bootstrapped_switch}

\subsubsection{Motivation}
Bootstrapping is necessary in order to avoid bandwidth modulation and harmonic distortion when charging the bottom plates of the CDAC up to $V_\mathrm{in}$. When an NMOS is used to charge up the CDAC,
the channel resistance $R_\mathrm{on}$ of the FET and the capacitance of the CDAC create a low-pass filter with cutoff frequency given by equation~\ref{eq:cutoff_frequency}.

\begin{equation}
    f_n = \frac{1}{2\pi R_\mathrm{on} C_\mathrm{DAC}} \label{eq:cutoff_frequency}
\end{equation}

The channel resistance $R_\mathrm{on}$ of an NMOS is given by equation~\ref{eq:nmos_resistance}.

\begin{equation}
    R_\mathrm{on} = \frac{1}{\mu_n C_{ox} \frac{W}{L} (V_{GS} - V_{th,n})} \label{eq:nmos_resistance}
\end{equation}

If the FET is gated by $V_\mathrm{DD}$, then $V_{GS} = V_\mathrm{DD} - V_\mathrm{in}$, meaning that if $V_\mathrm{in}$ changes, the bandwidth of the low-pass filter varies, attenuating signals inconsistently.
This unevenness results in harmonic distortion, lowering the effective resolution of the instrument. The bootstrapped switch \cite{abo1999bootstrap} combats this by lifting the gate voltage of the FET to $V_\mathrm{DD} + V_\mathrm{in}$, therefore 
fixing $V_{GS}$ at $V_\mathrm{DD}$, thus fixing $R_\mathrm{on}$ and the corresponding cutoff frequency of the transfer function.

\subsubsection{Topology and Operation}
The topology of the bootstrapped switch is shown in figure~\ref{fig:bootstrap}. It operates in two stages.

\begin{enumerate}
    \item When $\phi_s$ is low, the PMOS $Q_2$ is off while the NMOS $Q_5$ is on, therefore wiring the node X to ground. As such, the NMOS transistors $Q_4$ and $Q_0$ are off, while the PMOS $Q_1$ and the NMOS $Q_3$ are on. Therefore, there is a direct pathway from $V_\mathrm{DD}$ to ground through $C_\mathrm{boot}$, which charges up to $V_\mathrm{DD}$.
\item When $\phi_s$ turns on, transistors $Q_3$ and $Q_5$ turn off while transistor $Q_2$ turns on, connecting node X to the precharged top plate of $C_\mathrm{boot}$ and beginning to pull $V_X$ upward from ground. As $V_X$ rises, $Q_4$, whose gate is tied to X, turns on and connects the bottom plate of $C_\mathrm{boot}$ to $V_\mathrm{in}$; since $Q_3$'s turn-off has already isolated that bottom plate from ground, the capacitor's precharged $V_\mathrm{DD}$ is preserved across it and now rides on top of $V_\mathrm{in}$, settling $V_X$ at $V_\mathrm{DD} + V_\mathrm{in}$. The same rise in $V_X$ turns $Q_1$ off, disconnecting $C_\mathrm{boot}$'s top plate from the $V_\mathrm{DD}$ rail so the boosted voltage is not clamped, and turns $Q_0$ on, providing a pathway for $V_\mathrm{in}$ to flow into $V_{in,s}$ with its gate held at $V_X$.
\end{enumerate}

\subsubsection{Parameter Derivations}
Together with the CDAC, the bootstrapped switch enforces the three-phase behavior of the signal $\phi_s$. 
During phase 1, $\phi_s$ must be low so as to allow $C_\mathrm{boot}$ to charge. Then, $\phi_s$ must become high to turn the switch on and charge up the CDAC.
Finally, $\phi_s$ permanently turns off, disconnecting the CDAC from the bootstrapped switch which has accomplished its purpose by fixing charge onto the CDAC.

In totality, this means that $\phi_s$ must be a pulse that starts after a delay. The length of this pulse determines the sizing of the transistors and the value
of $C_\mathrm{boot}$. This length, in turn, is determined by the sample rate design target, 1 megasample per second.

A rate of 1 megasample per second implies a period of $\qty{1}{\micro s}$. Using a typical 70-30 split, with 30\% of the period allotted to sampling, the width of the pulse
is $\qty{300}{ns}$. During the width of this pulse, the CDAC must be charged sufficiently to avoid a rounding error. Practically, this means that the error
of the CDAC's voltage must be less than half of an LSB. The error of the CDAC's voltage, defined as the difference between $V_\mathrm{in}$ and $V_\mathrm{DAC}$, is given by equation~\ref{eq:cdac_error}.

\begin{equation}
    \text{Error} = V_\mathrm{in} \cdot e^{-t \, /  \, R_\mathrm{on} C_\mathrm{DAC}} \label{eq:cdac_error}
\end{equation}

Requiring this error to be less than the value of half of one LSB, $\text{Error} < \frac{V_\mathrm{ref}}{2^{N+1}}$, and assuming a worst-case scenario of $V_\mathrm{in} = V_\mathrm{ref}$, an expression for $R_\mathrm{on}$ can be derived.

\begin{equation}
    R_\mathrm{on} \le \frac{t}{C_\mathrm{DAC} \cdot (N+1) \cdot \ln 2} \label{eq:r_on}
\end{equation}

Substituting $t = \qty{300}{ns}$, $C_\mathrm{DAC} = \qty{25.6}{pF}$ and $N = 10$ into equation~\ref{eq:r_on}, $R_\mathrm{on}$ is found as $\qty{1.54}{\kilo \Omega}$. Applying the channel resistance equation of equation~\ref{eq:nmos_resistance},
with $V_{GS} = V_\mathrm{DD} = \qty{1.8}{V}$ due to bootstrapping, $\mu_n C_{ox} = \qty{270}{\frac{\micro A}{V^2}}$ and $V_{th,n} = \qty{0.48}{V}$ as set by Sky130, and $l = \qty{150}{nm}$ (the minimum allowable channel length in Sky130), % [CITATION NEEDED (optional): Sky130 device-parameter values used here and below]
the minimum value for $w$ is found as $\qty{273.8}{nm}$. Allowing for an ample margin, the transistor $Q_0$ was chosen to have a width of $\qty{1}{\micro m}$. Auxiliary transistors $Q_1$ and $Q_{3-5}$ were sized to match $Q_0$'s width for layout convenience,
while $Q_2$ was sized conservatively with a width double that of $Q_0$. All transistors have an identical length of $\qty{150}{nm}$.


Using these design parameters, a value of $C_\mathrm{boot}$ was selected that sufficiently minimizes droop. 
The voltage droop on $C_\mathrm{boot}$ due to charge sharing with the parasitic gate capacitance of $Q_0$, $C_\mathrm{gate}$, is given by equation~\ref{eq:droop}.

\begin{equation}
    \frac{\Delta V}{V_\mathrm{DD}} = \frac{C_\mathrm{gate}}{C_\mathrm{gate} + C_\mathrm{boot}} \label{eq:droop}
\end{equation}

Setting this quantity to be strictly less than 0.01,

\begin{gather}
    \frac{C_\mathrm{gate}}{C_\mathrm{gate} + C_\mathrm{boot}} < 0.01 \nonumber \\
    100 \cdot C_\mathrm{gate} < C_\mathrm{boot} + C_\mathrm{gate} \nonumber \\
    \boxed{C_\mathrm{boot} > 99 \cdot C_\mathrm{gate}} \label{eq:c_boot}
\end{gather}

The gate capacitance, $C_\mathrm{gate}$, is given by $C_\mathrm{gate} = wl \cdot C_{ox}$. Substituting $w = \qty{1}{\micro m}$, $l = \qty{150}{nm}$ and $C_{ox} = \qty{17}{\frac{fF}{\micro m^2}}$ as set by Sky130,
$C_\mathrm{boot} > \qty{252.45}{fF}$. To significantly exceed this mark, $C_\mathrm{boot}$ was chosen as $\qty{500}{fF}$. The final design choices are summarized in table~\ref{tab:sizing}.

When the clock is low, the pull-up network is active and the nodes P, Q, X and Y are pulled up to $V_\mathrm{DD}$. When the clock turns on, 
the pull-up network is disabled while the NMOS pull-down network comprised of M1, M2 and M7 is enabled, offering a pathway for the voltages at P and Q to discharge to ground. If $V_\mathrm{in1}$ exceeds $V_\mathrm{in2}$, the gate-source voltage and hence the rate of discharge at P is
greater than that of Q, and if the reverse is true ($V_\mathrm{in2} > V_\mathrm{in1}$), then Q discharges faster than P. Suppose that $V_\mathrm{in1} > V_\mathrm{in2}$.
Then, P discharges faster than Q, meaning that the NMOS M3 turns on before M4, offering a discharge pathway to ground for the voltage at X. 
Due to the cross-coupling between M3, M4, M5 and M6, X decreasing means that the gate voltage at M4 decreases, slowing the rate of discharge at Y.
Eventually, $V_X$ is low enough to enable M6 while disabling M4, hence pulling Y up to $V_\mathrm{DD}$. Therefore, if $V_\mathrm{in1} > V_\mathrm{in2}$, $V_Y$ settles at $V_\mathrm{DD}$
at steady state, while $V_X$ settles at 0. Following identical logic, in the reverse case where $V_\mathrm{in2} > V_\mathrm{in1}$, $V_{Y}$ settles at 0 while $V_{X}$ settles at $V_\mathrm{DD}$.

During operation, $V_\mathrm{in1} = V_\mathrm{ref} \, / \, 2$ while $V_\mathrm{in2} = V_\mathrm{DAC}$. As noted in subsection~\ref{subsection:cdac}, $V_\mathrm{DAC} = \frac{W}{1024} V_\mathrm{ref} + V_\mathrm{ref} \, / \, 2 - V_\mathrm{in}$ at the end of conversion. The condition for Y to be high is:

\begin{gather}
    V_\mathrm{in1} > V_\mathrm{in2} \nonumber \\
    V_\mathrm{ref} \, / \, 2 > V_\mathrm{DAC} \nonumber \\
    V_\mathrm{ref} \, / \, 2 > \frac{W}{1024} V_\mathrm{ref} + V_\mathrm{ref} \, / \, 2 - V_\mathrm{in} \nonumber \\
    0 > \frac{W}{1024} V_\mathrm{ref} - V_\mathrm{in} \nonumber \\
    \boxed{V_\mathrm{in} > \frac{W}{1024} V_\mathrm{ref}} \label{eq:comp_eqn}
\end{gather}

Therefore, when Y is high, the trial bit is asserted according to the SAR logic of section~\ref{section:architecture}. X and Y will always be complementary. The comparator outputs X and Y are fed to the R and S inputs respectively of an SR latch. The Q output of the latch, which tracks Y, is fed into the SAR logic block.

% ─────────────────────────────────────────────
\subsection{SAR Logic}
\label{subsection:sar_logic}

\subsubsection{Operation}

When \texttt{start} is asserted, signalling the start of a new conversion, the SAR logic clears its \texttt{result} and \texttt{cdac\_sw} registers, resets the bit index to 9, raises the internal \texttt{active} flag, and asserts \texttt{cdac\_sw[9]}, steering the MSB capacitor to $V_\mathrm{ref}$.

On each subsequent rising clock edge, the logic reads \texttt{comp\_in}, which tracks the SR latch output Q, and records the decision in \texttt{result[bit\_idx]}. The current trial bit in \texttt{cdac\_sw} is kept if \texttt{comp\_in} is high, or cleared if low. The next bit is then asserted in \texttt{cdac\_sw}, updating $V_\mathrm{DAC}$ for the following comparison as seen in figure~\ref{fig:cdac_cell}, and the bit index is decremented. This continues until bit 0 is evaluated, at which point \texttt{done} is asserted, \texttt{active} is cleared, and the block idles until the next \texttt{start} pulse.

\subsubsection{Implementation}
The SAR logic was written in Verilog RTL and verified behaviorally using Icarus Verilog. It was then synthesised to Sky130 high-density standard cells using Yosys, producing a gate-level SPICE netlist via Yosys's \texttt{write\_spice} command.

The Yosys-generated netlist required post-processing before it could be used in simulation. Yosys omits power pins (\texttt{VDD}/\texttt{GND}) from standard cell instantiations, and in some cases emits zero-volt voltage sources as net aliases rather than explicit connections. A Python script, \texttt{fix\_spice\_ports.py}, was written to correct both issues: it parses the Sky130 PDK SPICE library directly to recover the correct port order for each standard cell, inserts the missing power pins, resolves net aliases, and wraps the result in a \texttt{.subckt} block. The output, \texttt{sar\_logic\_fixed.spice}, is the netlist used in all subsequent simulations.

% ––––––––––––––––––––––––––––––––––––––––––––––––––––––––––––
\section{Integration \& Debugging}
\label{section:integration}

With all four blocks verified in isolation (Section~\ref{section:block_design}),
the remaining work was to close the loop and find why the full converter
did not reproduce the trajectory the behavioral model predicted. Most of
the design effort, and the most interesting findings, came from this
phase rather than from any single block.

% ------------------------------------------------------------
\subsection{Full-loop testbench}
\label{subsection:tb}

The integration testbench (\texttt{sar\_adc\_tb\_v2.spice}) wires the four
blocks into the conversion loop: the CDAC presents \texttt{vdac} to the
StrongARM comparator, which compares it against \texttt{vcm}; the
comparator outputs drive a two-NAND SR latch that holds the decision as
\texttt{comp\_in}; and the SAR logic consumes \texttt{comp\_in} to update
the ten switch controls \texttt{cdac\_sw[9:0]} for the next trial. A single
clock edge both captures the previous bit's decision and advances to the
next trial, so the loop needs only one clock domain.

\paragraph{Clocking.} Each conversion is ten cycles. The testbench runs
the clock at 500\,kHz (2\,\textmu s per cycle, 20\,\textmu s per
conversion), deliberately relaxed from the 1\,MS/s target rate so that
\texttt{vdac} settles fully on every trial and any error is attributable
to the circuit rather than to incomplete settling. Within a cycle, the
\texttt{cdac\_sw} update is followed by $\sim$990\,ns of settling before
\texttt{clkbar} goes high and the comparator evaluates, leaving
$\sim$1010\,ns for the comparator and SR latch to resolve before the next
rising clock edge reads \texttt{comp\_in}. This relaxed rate was a debugging choice rather than a fundamental limitation: the per-block timing budgets in Section~\ref{section:block_design}, most directly the bootstrapped switch's $R_\mathrm{on}$ sizing against a 300\,ns sampling window (Section~\ref{subsection:bootstrapped_switch}), were each derived against the full 1\,MS/s target. Closed-loop verification of the integrated converter at the full 1\,MS/s rate was not performed in this work and is identified as follow-on verification alongside the dynamic ENOB characterisation noted in Section~\ref{section:discussion}.

\paragraph{Cold-start initial conditions.} The single most consequential
testbench fix was forcing cold-start conditions with \texttt{.ic} and the
\texttt{UIC} flag on \texttt{.tran}. Without them, ngspice solves a DC
operating point at $t=0$ with \texttt{phi\_s} already mid-cycle, which
produces a non-physical startup transient on \texttt{vdac}: it overshoots
to $\sim$1.32\,V (roughly 400\,mV above its 0.9\,V reset target) and takes
hundreds of nanoseconds to settle, corrupting the early trials. The same
artifact appeared in two earlier block-pair testbenches before its origin
was understood, and it masked the real circuit behavior across several
sessions. The fix initialises \texttt{vdac} to its reset value
($V_\mathrm{cm}=0.9$\,V) and every CDAC bottom plate plus the sampled-input
node to $V_\mathrm{in}$, matching the state the sample phase will drive
them to anyway. 

\paragraph{Done-verified readout.} Rather than assuming \texttt{done}
asserts at a fixed time and reading the result there, the testbench
measures the actual assertion time with \texttt{.meas\ \dots\ WHEN} and
reads \texttt{result[9:0]} at 23\,\textmu s, with generous margin past the
$\sim$20.5\,\textmu s expected assertion. The measured assertion time then
serves as a sanity check that the read margin was in fact adequate. Every
\texttt{cdac\_sw} bit and \texttt{comp\_in} are also probed at full
resolution so the bit-by-bit decision sequence can be checked against the
precomputed ideal trajectory.

% ------------------------------------------------------------
\subsection{Isolation-first debugging methodology}
\label{subsection:isolation}

The first full-loop run at $V_\mathrm{in}=1.2$\,V returned a code roughly
77\,LSB below the expected value, an error which could not be attributed to rounding.
(The reference was later corrected to the true algorithmic target of 682,
making the figure $-76$\,LSB; the conclusion that something was seriously
wrong is unchanged.)

Rather than debug the closed loop directly, where any of four blocks could
be at fault, each block and then each pairwise combination was re-verified
in isolation:

\begin{itemize}
  \item \textbf{Switch alone} --- sampling accuracy versus input voltage.
  \item \textbf{Comparator alone} --- $\sim$350\,\textmu V input kickback,
        which is expected for a StrongARM topology and benign at this
        signal level.
  \item \textbf{CDAC alone} --- bottom-plate sampling and convert-phase
        switching.
  \item \textbf{CDAC + comparator chain} --- the analog half of the loop
        with no SAR logic in the path.
\end{itemize}

This narrowed the error to the sampling switch at high input voltages: the
comparator, CDAC, and digital logic were each correct, and the error scaled
with $V_\mathrm{in}$ in a way that pointed at the front-end switch.

% ------------------------------------------------------------
\subsection{The n-well bootstrap bug}
\label{subsection:nwell}

This was the most original finding and the one that took the most work to
pin down.

\paragraph{Symptom.} The bootstrapped switch undersampled by
$\sim$89\,mV at $V_\mathrm{in}=1.5$\,V, with negligible error at
$V_\mathrm{in}=1.2$\,V, a strongly input-dependent loss that grew toward
the top of the input range.

\paragraph{Wrong hypothesis first.} The initial theory was a channel
turn-off race on \texttt{XQ1}, the PFET that precharges the boost
capacitor's top plate. Because \texttt{XQ1}'s gate is the very node it is
trying to stop charging, an incomplete self-turn-off is a plausible
characteristic of this topology. A standalone leakage testbench with a
current-sense source in series with \texttt{XQ1}'s drain measured a real
$\sim$0.2\,mA transient at the boost transition, which appeared to confirm
the hypothesis.

\paragraph{Falsified by charge balance.} The hypothesis did not survive a
charge-balance check. Integrating that drain current over its conduction
window accounts for only $\sim$1\,fC, whereas the charge actually missing
from the boost capacitor was $\sim$394\,fC. The
measured channel current was real but roughly 400$\times$ too small to be
the dominant mechanism. Chasing that two-order-of-magnitude gap, rather
than accepting a plausible-looking number, is what found the actual bug.

\paragraph{Root cause: forward-biased PMOS body diode.} \texttt{XQ1} and
\texttt{XQ2} had their bulk pins tied to fixed $V_\mathrm{DD}$, the
correct default for ordinary PMOS, but wrong here because these two
devices' own source/drain nodes (referred to in the SPICE as \texttt{net1}, also known as X, and \texttt{net3}) are
deliberately boosted above $V_\mathrm{DD}$ during the sample phase. Once a
boosted node exceeds $V_\mathrm{DD}$ by more than a diode drop
($\sim$0.7\,V), the source/drain-to-bulk junction stops being reverse
biased and conducts as a forward diode, not a small leak but a hard
voltage clamp. The signature is unmistakable: \texttt{net3} settles at
$\sim$2.50\,V, almost exactly $V_\mathrm{DD}$ plus one diode drop. Adding
current-sense sources on the bulk pins themselves, a path the
drain-side sense source could not see, since bulk connects straight to the
$V_\mathrm{DD}$ rail externally, measured $\sim$382\,fC/cycle at
$V_\mathrm{in}=1.5$\,V, accounting for $\sim$97\% of the deficit.

The numbers are self-consistent. The boost node should reach
$V_\mathrm{in}+V_\mathrm{DD}=3.3$\,V at $V_\mathrm{in}=1.5$\,V but clamps
at 2.5\,V, a 0.8\,V shortfall. On the 500\,fF boost capacitor that
corresponds to $0.8\,\mathrm{V}\times 500\,\mathrm{fF}=400$\,fC of charge
that never makes it onto the node, within measurement error of the
394\,fC found missing.

\paragraph{Fix.} A new \texttt{nwell\_switch} (figure~\ref{fig:nwell_fix}) subcircuit generates a
floating bias node that always tracks whichever is higher,
$V_\mathrm{DD}$ or \texttt{net1}: $V_\mathrm{DD}$ during precharge,
\texttt{net1} during boost. The bulk pins of \texttt{XQ1} and \texttt{XQ2}
route through this node instead of fixed $V_\mathrm{DD}$, so the junction
never forward-biases. The subcircuit's own two internal devices have their
bulks tied to its output node, so the fix does not reintroduce the same
problem on itself.

\begin{figure}[H]
    \centering
    \includegraphics[width=0.85\textwidth]{figures/bootstrap_fix_waveform.png}
    \caption{Bootstrapped switch output at $V_\mathrm{in}=1.5$\,V before and
    after the n-well fix. Pre-fix (red): the forward-biased body diode clamps
    the boost node, limiting \texttt{VOUT} to $\approx 1.40$\,V with a
    $\sim$450\,ns settling tail into the second sample window. Post-fix
    (blue): \texttt{VOUT} reaches $V_\mathrm{in}$ within the first sample

    \label{fig:bootstrap_fix_waveform}

dropped from $\sim$89\,mV to 0.94\,\textmu V (figure~\ref{fig:bootstrap_fix_waveform}), and the bulk-junction current
collapsed from $-382$\,fC to $+0.15$\,fC per cycle. Re-running the full
ADC gave $-6$, $-11$, and $-54$\,LSB at $V_\mathrm{in}=0.6$, $1.2$, and
$1.5$\,V respectively; the dominant front-end loss removed, leaving a
smaller residual at high input that is the subject of the next subsection.

\subsection{Residual error: top-plate reset feedthrough}
\label{subsection:feedthrough}

The $-54$\,LSB at $V_\mathrm{in}=1.5$\,V that remains after the bootstrap
fix is a separate mechanism. At the sample-to-convert transition, the
top-plate reset switch \texttt{XTOP\_n} couples charge onto \texttt{vdac}
through its gate-drain overlap capacitance, kicking the node away from its
reset value just as the conversion begins.

Isolated testbenches characterised the kick as a function of input
voltage:

\begin{table}[H]
    \centering
    \caption{Residual error on $V_\mathrm{DAC}$ as a function of input voltage}
    \label{tab:residual_error}
    \begin{tabular}{cc}
        \toprule
        $V_\mathrm{in}$ (V) & Kick on \texttt{vdac} \\
        \midrule
        0.3 & $\sim$0.6\,mV \\
        0.6 & $\sim$4.8\,mV \\
        0.9 & $\sim$9\,mV \\
        1.2 & $\sim$15\,mV \\
        1.5 & $\sim$122--160\,mV (with a $\sim$450--500\,ns settling tail) \\
        \bottomrule
    \end{tabular}
\end{table}

The kick grows nonlinearly with $V_\mathrm{in}$ and develops a long
settling tail near the top of the range. Because the error is strongly
input-dependent rather than a fixed offset, it cannot be removed by a
single static digital calibration constant. Known mitigations (a dummy
top-plate switch for charge cancellation, or a separate reset phase that
lets \texttt{vdac} settle before the comparator looks at it) were
identified but not implemented, since the converter already meets its
functional goals and physical layout is the next priority rather than
squeezing the last few LSB of high-input linearity.

\label{section:static}

\subsection{DC Sweep Methodology}

Static linearity was characterised by sweeping a DC input voltage from 0 to 1.8\,V and recording the 10-bit output code at each point. The sweep was implemented in \texttt{sweep\_adc.py}, which spawns a separate ngspice process per input voltage rather than running a monolithic multi-point simulation, which would have exhausted ngspice's internal memory.

Input voltages were stepped in 10\,mV increments, giving a theoretical 181 points over 0--1.8\,V. Of these, 141 points returned valid \texttt{done}-asserted conversions, with the other 40 failing to complete before the self-imposed 180-second time limit. The operating range was auto-detected as 0.50--1.39\,V by \texttt{find\_operating\_range.py}, which identifies the contiguous span of codes whose conversion error stays within $\pm$35\,mV of the ideal; points outside this range reflect the bootstrapped switch failing to fully charge the CDAC at low input voltages, and CDAC top-plate feedthrough saturating the output at high voltages. Fig.~\ref{fig:operating_range} shows the full-range error plot used for this detection.

\begin{figure}[H]
    \centering
    \includegraphics[width=\textwidth]{figures/operating_range_plot.png}
    \caption{ADC conversion error over 0--1.8\,V. The shaded region (0.50--1.39\,V) is the auto-detected operating range, bounded by a $\pm$35\,mV threshold. The monotone ramp below 0.5\,V reflects bootstrapped switch underdriving; the steep negative error above 1.39\,V is top-plate feedthrough from \texttt{XTOP\_n} growing nonlinearly as $V_\mathrm{in}$ approaches $V_\mathrm{DD}$.}
    \label{fig:operating_range}
\end{figure}

\subsection{INL Results}

INL was computed by fitting a best-fit line to the transfer function over the valid operating range and measuring the deviation of each code transition from the ideal. The raw and smoothed INL curves are shown in Fig.~\ref{fig:inl}.

\begin{figure}[H]
    \centering
    \includegraphics[width=\textwidth]{figures/inl_plot.png}
    \caption{Raw and smoothed INL over the valid input range (0.50--1.39\,V). The diagonal line visible in the raw curve near 0.79--1.01\,V is an interpolation artefact caused by missing simulation points in that window, not a real feature of the transfer function. The $+$5\,LSB peak in the smoothed curve near 0.75\,V is caused by top-plate reset feedthrough from \texttt{XTOP\_n}, discussed further in Section~\ref{section:discussion}.}
    \label{fig:inl}
\end{figure}

The raw INL has a peak of $\pm$13\,LSB and an RMS of 9.21\,LSB. The dominant contribution to this figure is a measurement grid artefact: a 10\,mV step corresponds to approximately 5.6 output codes at 10-bit resolution over 0.89\,V of valid range, so the transfer function is sampled far below Nyquist for code-level detail. Each transition between grid points is assigned to the nearest sampled code, introducing a staircase quantisation error that manifests as high-frequency oscillation in the INL curve and inflates the RMS well beyond the true device nonlinearity. Applying a 7-point moving average suppresses this artefact and gives a smoothed INL of $\pm$5\,LSB peak and 1.90\,LSB RMS.

\subsection{Static ENOB}

A static ENOB figure was derived from the INL by treating INL error as noise. The signal power of a full-scale sinusoid over a 10-bit ADC is $(2^{10})^2/8$ in units of LSB$^2$; the noise power is taken as $\sigma_{\mathrm{INL}}^2 + 1/12$, where the $1/12$ term accounts for ideal quantisation noise. Using the smoothed INL RMS of 1.90\,LSB:

\begin{equation}
    \mathrm{SNDR} = 10\log_{10}\!\left(\frac{(2^{10})^2/8}{\sigma_{\mathrm{INL}}^2 + 1/12}\right) = 45.50\,\mathrm{dB}
\end{equation}

\begin{equation}
    \mathrm{ENOB} = \frac{\mathrm{SNDR} - 1.76}{6.02} = 7.27\,\mathrm{bits}
\end{equation}

These results are summarised in the title bar of Fig.~\ref{fig:inl}. This falls short of the 9-bit target.

This static figure should be read in both directions rather than taken at face value. On one hand it is a lower bound on the converter's true dynamic performance, since a DC sweep cannot capture sampling jitter or input bandwidth limitations, both of which would only erode ENOB further under a real time-varying signal. On the other hand it is simultaneously an upper bound, for the opposite reason: the dominant error source in this measurement, the top-plate feedthrough kick above roughly 1.3\,V (Section~\ref{subsection:feedthrough}), which contributes the $\pm$13\,LSB raw peak, is itself an artifact of how a DC sweep is taken. Each point in the sweep is a separate sample-and-convert cycle, so every point pays the kick once; a genuine sine-wave input sampled at 1\,MS/s would not necessarily revisit the same high-voltage codes with the same per-cycle penalty, and the kick's settling tail would interact with the actual sampling instants rather than with an artificially quiescent DC level. The 7.27-bit figure is therefore neither a guaranteed floor nor ceiling on dynamic performance; it reflects a specific, identified error mechanism evaluated under a measurement methodology that this mechanism is somewhat pessimistic to. A sine-wave FFT test would resolve this ambiguity directly and has not yet been performed due to processing power constraints.

\section{Discussion}
\label{section:discussion}

\subsection{Performance Against Specification}

The design targeted $\geq$9-bit ENOB at 1\,MS/s on a 1.8\,V supply. The measured static ENOB of 7.27 bits falls short by approximately 1.7 bits. The shortfall has a single identified root cause: the top-plate reset switch \texttt{XTOP\_n} injects a feedthrough kick onto the CDAC top plate at the end of the sampling phase via gate-drain overlap capacitance ($C_\mathrm{GDO} = 2.449 \times 10^{-10}$\,F/m). The kick is input-dependent and grows nonlinearly as $V_\mathrm{in}$ approaches $V_\mathrm{DD}$, reaching approximately 15\,mV at $V_\mathrm{in} = 1.2$\,V and 122--160\,mV at $V_\mathrm{in} = 1.5$\,V with a settling tail of 450--500\,ns. Because the error magnitude is a nonlinear function of the input, it cannot be removed by static digital calibration.

The 1\,MS/s sample-rate target was met at the level of individual block timing budgets (Section~\ref{section:block_design}) but was not verified end-to-end: all integration and static characterisation in this report used a relaxed 500\,kHz testbench clock, chosen so that settling error could be ruled out as a source of conversion error during debugging. Running the integrated converter at the full clock rate, where comparator regeneration and SAR logic propagation must also complete within the tighter timing budget rather than just the sampling switch, is necessary before the 1\,MS/s specification can be considered fully demonstrated.

\subsection{Operating Range}

The recommended input range is 0.50--1.39\,V, approximately 50\% of the full supply span. The lower bound is set by bootstrapped switch headroom: as $V_\mathrm{in}$ approaches ground, the gate overdrive on the main switch transistor collapses and the switch can no longer fully charge the CDAC within the sampling window. The upper bound is set by the feedthrough nonlinearity described above. Specifying an inset operating range is standard practice for SAR ADCs; the full-scale input range nominally matching the supply rails does not imply usable linearity across that entire range.

\subsection{Mitigation Paths}
\label{subsection:mitigation}

Several standard techniques would reduce or eliminate the feedthrough error but were not implemented, as they fall outside the scope of this project:

\begin{itemize}
    \item \textbf{Dummy top-plate switch:} A half-sized switch driven by the complement of the reset clock can cancel the kick by injecting an equal and opposite charge. This is the most common first-order fix.
    \item \textbf{Separate reset phase:} Decoupling the top-plate reset from the sampling phase gives the kick time to settle before conversion begins, at the cost of a more complex clocking scheme.
    \item \textbf{Digital background calibration:} The input-dependent error profile could in principle be characterised and subtracted digitally, but the nonlinearity near $V_\mathrm{DD}$ is large enough that this would require significant calibration range and does not address the settling tail within a 1\,MS/s conversion window.
\end{itemize}

\subsection{Physical Layout}

Magic layout was considered, but not pursued. The project scope is full-custom mixed-signal design at the schematic and simulation level: behavioral modelling, transistor sizing, block-level and system-level SPICE simulation, and static characterisation. Physical implementation would be the natural next step to demonstrate a complete design-to-GDS flow, but it does not add further insight into the analog design decisions documented here.

Expressed in LSB ($V_\mathrm{LSB} = \qty{1.76}{mV}$ at 10-bit resolution), the kick is below 1\,LSB across most of the input range and only becomes significant near $V_\mathrm{DD}$: 15\,mV at $V_\mathrm{in}=1.2$\,V is $\sim$8.5\,LSB, while 122--160\,mV at $V_\mathrm{in}=1.5$\,V is 70--91\,LSB. The static ENOB figure is therefore set almost entirely by the behavior above approximately 1.3\,V; restricting the operating range further or applying the mitigations in Section~\ref{subsection:mitigation} would be expected to recover most of the missing 1.7 bits, since the rest of the transfer function (Section~\ref{section:static}) already sits close to the ideal characteristic.

\section{Conclusions}
\label{section:conclusions}

This project demonstrated a complete mixed-signal design flow for a 10-bit 1\,MS/s SAR ADC in the SkyWater Sky130 130\,nm open-source PDK, from behavioral modelling in Python through transistor-level SPICE simulation and static characterisation. All four blocks (bootstrapped switch, charge-redistribution CDAC, StrongARM comparator, and SAR control logic) were designed, sized from first principles, and verified individually before system-level integration.

The SAR logic was synthesised from a Verilog behavioral description using Yosys and integrated as a standard-cell netlist alongside the full-custom analogue blocks, demonstrating a mixed-signal methodology applicable to real tape-out flows. A non-trivial transistor-level bug, PMOS n-well forward biasing under bootstrapped gate drive, was identified and resolved through charge-balance analysis rather than empirical tuning, accounting for 97\% of the observed conversion error once correctly attributed. The static characterisation yielded 7.27-bit ENOB against a 9-bit target, with the shortfall fully explained by a characterised top-plate feedthrough mechanism.

Falling short of the 9-bit target is, on its own, less significant than how that shortfall was reached: every discrepancy between the behavioral model and the transistor-level result was traced to a specific, physically identified mechanism rather than left as an unexplained gap, and the n-well bug in particular was found by following a charge-balance inconsistency the initial hypothesis could not account for. That methodology, isolating each block, treating disagreements between simulation and prediction as leads rather than noise, and quantifying root causes before attempting fixes, is what the remainder of the design flow (synthesis, layout, and tapeout) would build on.
